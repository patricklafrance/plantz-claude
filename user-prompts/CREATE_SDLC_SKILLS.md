I want you to create a collection of agent skills to will map different profile of the Software Development Lifecycle. The goal is to build a collection of agent skills that will be used by an orchestor agent and different subagents to develop a feature.

All agent skills must be created using the `skill-creator` agent skill and 2 subagents to assist and challenge each others.

The agent skills to create are:

- `plantz-sdlc-orchestrator`: An agent skill for the orchestrator agent that will lead and be accountable for the development of the feature. The orchestrator agent will orchestrator the different steps of the process and coordinate subagents.
- `plantz-sdlc-plan`: An agent skill to plan the development of the code of the feature. The planning subagents will elaborate the plan the output it in a file.
- `plantz-sdlc-code`: An agent skill to execute the plan. The coding subagents will load the plan file and execute the plan.
- `plantz-sdlc-test`: An agent skill to test the code generated by the coding subagents. With the help of the orchestrator agent, the testing subagents will look for issues in the code and output their findings in a file.
- `plantz-sdlc-document`: An agent skill to update the repository agent documentation. Documentarian subagents will look at the plan and the changes and identify drifts in the existing documentation. If any drifts are found, they will fix them.
- `plantz-sdlc-release`: An agent skill to commit the code and open and monitor the pull request.

## plantz-sdlc-orchestrator

The `plantz-sdlc-orchestrator` agent skill purpose is to lead the development of the feature, spawn subagents, and coordinate their work. This is the entry point for developing a new feature.

Through the whole process, this agent skill will do in the following order these steps (each step must be done before proceeding to the next one):

- Generate a hash for this run and pass it to subagents.
- Create a branch
- Spawn two subagents that will use the `plantz-sdlc-plan` agent skill to plan the development of the feature.
- Spawn two subagents that will use the `plantz-sdlc-code` agent skill to implement the plan.
- Spawn two subagents that will use the `plantz-sdlc-test` agent skill to test the generated code and iterate with the existing coding subagents to fix any issues found by the testing subagents.
- Simplify the generated code using the `/simplify` skill.
- Spawn two subagents that will use the `plantz-sdlc-document` agent skill to look for drifts between the code and agentic documentation and update if anything is found.
- Spawn a subagent that will use the `plantz-sdlc-release` agent skill to commit the changes, open the pull request and monitor the pull request.
- Clean up the `./tmp/runs/[run-hash]/` folder.

This agent skill is the glue between every subagents that will take part in the SDLC.

## plantz-sdlc-plan

The `plantz-sdlc-plan` agent skill purpose is use plan mode to draft the technical approach to implement the feature. The goal is to output a `./tmp/runs/[run-hash]/plan.md` file including the technical approach to implement the feature. The `run-hash` will be provided by the orchestrator agent (or the consumer in this case).

The following instructions must be followed when drafting the plan:

- Before proceeding, look for relevant records in [ADR](./agent-docs/adr/index.md).
- Before proceeding, look for relevant records in [ODR](./agent-docs/odr/index.md).
- Before proceeding, read [ARCHITECTURE.md](./agent-docs/ARCHITECTURE.md).
- The code must be in React and TypeScript. To help with that, the agent must load the `workleap-react-best-practices` agent skill.
- The code must use Tailwind for styling.
- The code must be WCAG AA compliant. To help with that, the agent must load the `accessibility` agent skill.
- The code must support light and dark color scheme (when creating the skill, move the information from `./agent-docs/references/color-mode.md` to this agent skill and, if there's nothing left, delete the file).
- The code must support phone, tablet and desktop devices (when creating the skill, move the information from `./agent-docs/references/responsive-layout.md` to this agent skill and, if there's nothing left, delete the file).
- The code must use [Tanstack DB](https://tanstack.com/db/latest) for persistence. Currently, it should only support [local storage collections](https://tanstack.com/db/latest/docs/collections/local-storage-collection) (when creating the skill, move the information from `./agent-docs/references/tanstack-db.md` to this skill and, if there's nothing left, delete the file).
- The code must use [shadcn](https://ui.shadcn.com/) v4 with Base UI, Tailwind and PostCSS for components (when creating the skill, move the information from `./agent-docs/references/shadcn.md` to this skill and, if there's nothing left, delete the file).
- The agent must generate [Storybook](https://storybook.js.org/) stories with multiple variants for every components (when creating the skill, move the information from `./agent-docs/references/storybook.md` to theisskill and, if there's nothing left, delete the file).
- The agent must load the `frontend-design` agent skill to help design any feature.
- If the agent have to scaffold a new domain module, the agent must load and use the `plantz-scaffold-domain-module` agent skill to scaffold the domain.
- If the agent have to scaffold a new Storybook, the agent must load and use the ` plant-zscaffold-domain-storybook` agent skill to scaffold the Storybook.
